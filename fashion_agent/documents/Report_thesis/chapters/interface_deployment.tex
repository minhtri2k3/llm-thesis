\textbf{Why store \texttt{year\_of\_birth} and \texttt{gender}?}
\begin{itemize}
    \item \textbf{Evaluation axis for thesis analysis}: demographics are used to segment satisfaction and behavior metrics by age/gender cohorts (cross-age and cross-gender analysis).
    \item \textbf{Controlled personalization (not hard enforcement)}: \texttt{gender} works with a gender-hint flag (\texttt{gender\_hint\_enabled}) as an A/B switch, allowing comparison between hint-enabled and hint-disabled runs.
    \item \textbf{Safety/quality monitoring}: analytics endpoint \path{/api/analytics/gender-ab} computes Gender-Appropriate Selection (GAS) to quantify match vs mismatch selections by orchestration mode.
    \item \textbf{Data quality}: API-level validation restricts \texttt{year\_of\_birth} to a valid range and \texttt{gender} to \texttt{male/female} values.
\end{itemize}
