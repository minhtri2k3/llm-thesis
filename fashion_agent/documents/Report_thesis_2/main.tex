\chapter*{Abstract}
\addchaptertoc{Abstract}

\noindent
This thesis presents \textit{Clothie}, an intelligent fashion search and recommendation system designed to achieve high precision and rapid response times through the synergy of a modular processing pipeline and Large Language Model (LLM) orchestration. Unlike conventional RAG systems that handle structured text retrieval, \textit{Clothie} tackles the inherent subjectivity of image-based fashion discovery. The system specifically addresses two fundamental "uncertainty" challenges:

\noindent
(i) The Specification Gap: when a user identifies a specific product category (e.g., "a jacket") but lacks the precise attributes—such as material, fit, or stylistic nuances—required for an accurate search. \textit{Clothie} acts as a proactive agent to clarify these hidden preferences, ensuring the retrieved results align with the user's exact vision.

\noindent
(ii) The Contextual Indecision: when a user has a situational need (e.g., "attending a summer wedding") but is uncertain about which type of apparel to seek. In these cases, the system leverages LLM reasoning to bridge the gap between a vague scenario and concrete, contextually relevant product suggestions.

\noindent
The work utilizes the public \textit{Clothing Dataset Full} from Kaggle, where each item is enriched with vision-language descriptors via Google Gemini~2.5~Flash. The implementation comprises a robust Python-based pipeline, evolving from a hybrid retrieval baseline (\textit{Hybrid RAG}) to a single-LLM, intent-first deterministically routed agent (\textit{Agent~v2.0}). The agent uses Gemini~2.5~Flash as the orchestrator: it performs intent classification with six-slot extraction in one call, applies a deterministic clarification gate when slots are insufficient, and otherwise invokes a seven-stage tool chain (BM25, FashionSigLIP image and text vectors, RRF fusion, fuzzy filtering, and a BGE Reranker v2-m3 cross-encoder). Each turn issues only one or two LLM calls --- replacing a multi-step ReAct loop --- to maximise retrieval accuracy under tight cost and latency budgets.

\noindent
Empirical evaluation against the deployed PostgreSQL telemetry shows that the agent issues exactly one or two LLM calls per turn --- confirming that no multi-step ReAct loop runs in production --- and that users actively engage with the displayed results in PATH~1, with a click-through rate of 31.9\% and a cart-add rate of 8.3\% of impressions. These signals, together with the intent-first clarification logic of the single-LLM agent detailed in \textbf{Methodology (Chapter 3)}, demonstrate that the agent can navigate ambiguous fashion queries while keeping per-turn computational cost bounded. Qualitatively, the agent resolves uncertain user states through proactive clarification and maps complex lifestyle contexts to high-quality product suggestions.
