## Actual Architecture (source of truth: `agent/fashion_agent.py` line 7)

```
Architecture (v2 — direct routing, no ReAct loop):
  1. classify_intent       → 1 LLM call (intent + slots)
  2. slot gate             → template clarification (0 LLM)
  3. _route_and_execute    → deterministic search routing (0 LLM)
  4. _synthesize_response  → Gemini synthesis (1 LLM call)
```

**Total LLM calls per turn: 2 (happy path) or 1 (clarification path).**

## New End-to-End Diagram (TikZ)

Replace the old loop diagram (Fig 3) with a **linear 6-step flow**:

```
User Query
    │
    ▼
[Step 0] Session Setup (0 LLM)
  → create/load session; log user message
  → confirm/reject keyword check → early exit
    │
    ▼
[Step 1] Intent Classification (1 LLM)
  → intents: text_search, follow_up, outfit_request,
             out_of_scope, product_select, view_selections
  → ExtractedSlots: category, color, fabric, fit, construction, aesthetic
  → EARLY EXIT: out_of_scope / product_select / view_selections
    │
    ▼
[Step 2] Slot Gate (0 LLM — template)
  text_search:
    → merge slots (TTL cache 30min)
    → check: category + color + (fabric OR fit)?
    → if missing AND turns < 3 → template question → EXIT
    → compose_refined_query_from_slots()
  follow_up: → merge slots, compose query
  unclear (conf<0.6): → check_clarification()
    │
    ▼
[Step 3] Log + Preferences (0 LLM)
  → log_query() → PostgreSQL
  → get_preferences()
    │
    ▼
[Step 4] Hybrid Search (0 LLM — local models)
  → BM25 top-5 + Image Vec ANN top-5 + Text Vec ANN top-5
  → Dedup per source
  → 3-way RRF (BM25×2.5, text_vec×1.5, img_vec×1.0)
  → Filter scoring / Soft RapidFuzz filter
  → BGE Reranker + score blend
  → Score threshold ≥ 0.25 → top-6
    │
    ▼
[Step 5] Gemini Synthesis (1 LLM — streaming)
  → ThinkingToken (hidden) + ResponseToken (streamed)
  → token usage → PostgreSQL
  → save reply to memory
    │
    ▼
Agent Response (SSE: thinking_start → tokens → products → done)
```

## New Design Decision D7

**Replace** the ReAct loop decision with:

> **D7: Slot gate over LLM clarification.**
> Rather than asking Gemini to generate clarifying questions (an extra LLM call per ambiguous turn), the system uses a template-based slot completeness checker. `category + color + (fabric OR fit)` must be filled before a search executes. Missing slots trigger pre-written template questions — zero extra API calls, deterministic latency, guaranteed termination after max 3 clarification turns.
